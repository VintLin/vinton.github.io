原教程： ChatGPT Prompt Engineering for Developers

中英字幕： 《ChatGPT提示工程》非官方版中英双语字幕

本文是对ChatGPT Prompt Engineering for Developers课程内容的总结归档，另外课程中的英文Prompt也翻译为中文，方便理解和学习。

原则一：清楚和具体的指示

策略1：使用分隔符

  • 三重引号: """
  • 三重反引号: ```
  • 三重破折号: ---
  • 尖括号: <>
  • XML标签: <tag> </tag>

策略2：要求结构化输出

  • JSON, HTML

策略3：检查条件是否满足检查完成任务所需的假设

策略4：少样本提示。

原则二：给模型时间”思考”

策略1：明确完成任务所需的步骤

要求以指定格式输出

策略2：在草率得出结论之前，要求模型自行找出解决方案

注意，学生的解答实际上是不正确的。

我们可以通过指导模型先自己解决问题来修复这个问题。

模型的缺陷：

幻觉

对提示进行迭代（Iterative Prompt Development）

🎯迭代流程：想法 → 实现（代码/数据） → 实验结果 → 错误分析 → 重复

提示指南

  • 清晰且具体
  • 分析为什么结果没有得到期望的输出
  • 精炼想法和提示
  • 重复

迭代过程

  • 尝试某些操作
  • 分析结果为何不符合你的期望
  • 明确指示，给予更多的思考时间
  • 用一批例子来精炼提示

以下方提示举例：

错误分析

问题 1. 文本过长

  • 限制单词/句子/字符的数量。

问题 2. 文本关注了错误的细节

  • 让它关注对目标受众相关的方面。

问题 3. 描述需要一个尺寸表

  • 要求它提取信息并将其组织成表格。

  • 得到的总结中包括与关注主题无关的话题。
